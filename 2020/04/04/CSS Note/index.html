一、块（行）、内联（行内）、内联块（行内块）

1、块元素 / 行元素

布局中常用的标签如：div、p、ul、li、h1~h6、dl、dt、dd 等在布局中的行为：
 1. 支持全部的样式
 2. 如果没有设置宽度，默认的宽度为父级宽度的100%
 盒子占据一行，即使设置了宽度

例：


2、内联块元素 / 行内块元素

布局中常用的标签如：a、span、em、b、strong、i 等，在布局中的行为：
 1. 不支持宽width、高height、margin-top、margin-bottom、padding-top、padding-bottom
 2. 宽高由内容决定
 3. 盒子并在一行
 4. 代码换行，盒子之间会产生间距
 5. 子元素是内联元素，父元素可以用 text-align 属性设置子元素水平对齐方式，用 line-height 属性设置子元素垂直对齐方式（line-height 与 height 的 px 相同即居中）

解决内联元素之间间隙的方法：
 1. 去掉内联元素之间的换行
 2. 将内联元素的父级设置 font-size 为 0 ，内联元素自身再设置 font-size


3、内联块元素 / 行内块元素

img 和 input 元素的行为类似这种元素，但是归类于内联元素，可以用 display 属性将 块元素 或者 内联元素 转化成 内联块元素 ，在布局中表现的行为：
 1. 支持全部样式
 2. 如果没有设置宽高，宽高由内容决定
 3. 盒子并在一行
 4. 代码换行，盒子会产生间隙
 5. 子元素是内联元素，父元素可以用 text-align 属性设置子元素水平对齐方式，用 line-height 属性设置子元素垂直对齐方式（line-height 与 height 的 px 相同即居中）

这三种元素，可以通过 display 属性来相互转化，不过 块元素 用得较多，经常把 内联元素 转化为 块元素 ，少量转化为 内联块元素 ，而需要使用 内联元素 时，直接使用 内联块元素 ，不用 块元素 转化。

display 属性：用来设置元素的类型及隐藏的，常用的选项有：
 1. none ：元素隐藏且不占位置
 2. block ： 元素以 【块元素】 显示
 3. inline ： 元素以【内联元素】 显示
 4. inline-block ： 元素以【内联块元素】 显示

四、浮动

1、文档流

文档流指的是：盒子按照 html 标签编写的顺序依次从上到下，从左到右排序。块元素占一行，行内元素在一行之内从左到右排列，先写的先排列，后写的排在后面，每个盒子都占据自己的位置。


2、浮动特性

 1. 浮动元素有左浮动 float:left ，和右浮动 float:right 两种。

 2. 浮动元素会向左或向右浮动，碰到父元素边界、浮动元素、未浮动的元素，才停下来。

 3. 相邻浮动的块元素可以并在一行，超出父级宽度就换行。

 4. 浮动让 行内元素 或 块元素 自动转化为 行内块元素 （但是与直接 display:inline-clock 并不是很相同，直接转化后，存在换行后的元素间隔能撑开父元素，但是浮动的转化就不可）

 5. 【浮动元素】 后面 【没有浮动的元素】 会占据 【浮动元素】 的位置，【没有浮动的元素】 内的文字会避开 【浮动元素】，形成文字饶图的效果。

五、定位

可以使用 CSS 的 position 属性来设置元素的定位类型，设置项如下：

  1. relative 生成 【相对】 定位，，元素所占据的文档流的位置不变，元素本身相对文档流的位置（原来自身）进行偏移。

  2. absolute 生成 【绝对】 定位，元素脱离文档流，不占据文档流的位置，可以理解为漂浮在文档流的上方，相对于上一个设置了 【relative相对】 或者 【absolute绝对】 或者 【fixed固定】 定位的父级元素来进行定位，若找不到，则相对于 body 元素进行定位。

  3. fixed 生成 【固定】 定位，元素脱离文档流，不占据文档流的位置，相对于浏览器窗口进行定位。

  4. static 默认值，没有定位。

  5. inherit 从父元素继承 position 属性的值


1、 定位元素的特性

【absolute】 定位 和 【fixed】 定位的块元素和行内元素会自动转化为行内块元素


2、 定位元素的层级

定位元素是浮动在正常的文档流之上的，可以用 z-index 属性来设置元素的层级，哪个值最大，就出现在最上面。

七、box-shadow

box-shadow 属性用于在元素的框架上添加阴影效果，该属性可设置的值包括X轴偏移、Y轴偏移、阴影模糊半径、阴影扩散半径，和阴影颜色。如果元素同时设置了 border-radius ，阴影也会有圆角效果。

box-shadow : offset-x offset-y blur spread color inset

  1. offset-x 和 offset-y ：<offset-x> 设置水平偏移量，正值阴影位于元素右边，负值在左； offset-y 设置垂直偏移量，正值阴影位于元素下方，负值在上。如果两者都是0，则阴影位于元素后面，这时如果设置了 blur 或 spread 则有模糊效果。

  2. blur : 模糊距离。值越大，模糊面积越大，阴影就越大越淡。不可为负值。

  3. spread ：阴影尺寸。值为正时，阴影扩大；为负时，阴影缩小。

  4. color ：阴影颜色

  5. inset ：没有指定 inset ，默认阴影在外边框外；使用 inset 关键字会使得阴影在盒子内部，看起来就像是内容被压低了，此时阴影会在背景之上，内容之下。

九、弹性盒子（Flex Box）

Flex Box 是 CSS3 的一种布局模式，是一种当页面需要适应不同屏幕大小以及设备类型时确保元素拥有恰当的行为的行为布局方式。

可以对一个容器内的子元素进行排列、对齐和分配空白空间。

设置 display 属性为 flex 或 inline-flex 即可以定义为弹性容器，包含在容器内的称为弹性项。


9.1、 弹性容器

对弹性容器可以设置以下的六个属性：

  1. flex-direction
  2. flex-wrap
  3. flex-flow
  4. justify-content
  5. align-items
  6. align-content

9.1.1 flex-direction

flex-direction 决定了弹性项在容器内的主轴方向。

四个值分别表示：

  • column_reverse : 垂直，弹性项的起点在下沿。
  • column : 垂直，弹性项的起点在上沿。
  • row(默认值) : 水平，弹性项的起点在左边。
  • row-reverse : 水平，弹性项的起点在右边。

如图：


9.1.2 flex-wrap

flex-wrap 指定了弹性项在弹性容器内的换行方式。

三个值分别表示：

  • nowrap(默认值) : 不换行。
  • wrap ： 换行，第一行在上方。
  • wrap-reverse : 换行，第一行在下方。

如图：


9.1.3 flex-flow

flex-flow 是 flex-direction 和 flex-wrap 的结合。


9.1.4 justify-content

justify-content 定义弹性项在水平上（x轴）的对齐方式。

五个值分别表示：

  • flex-start(默认值) : 左对齐依次排放。
  • flex-end : 右对齐。
  • center : 弹性项居中紧挨着填充。（如果剩余的自由空间是负的，则弹性项将在两个方向上同时溢出）。
  • space-between : 两端对齐，如果只有一个弹性项或剩余空间为负，则同 flex-start 相同；正常为：第一个弹性项贴齐左边，最后一个弹性项贴齐右边，然后剩余的弹性项分布在该行上，相邻的间隔相等。
  • space-around : 弹性项平均分布在该行上，两边留有一半的间隔空间。如果剩余空间为负或者只有一个弹性项，则该值等同于center。否则，弹性项沿该行分布，且彼此间隔相等（比如是20px），同时首尾两边和弹性容器之间留有一半的间隔（1/2*20px=10px）。

如图：


9.1.5 align-items

align-items 定义了弹性项在垂直上(y轴)的对齐方式。

五个值分别表示：

  • flex-start : 弹性项的纵轴对齐容器上方纵轴。
  • flex-end : 弹性项的纵轴对齐容器下方的纵轴。
  • center : 弹性项在容器y轴上居中放置。（如果荣容器的尺寸小于弹性项的尺寸，则会向两个方向溢出相同的长度）。
  • baseline : 弹性项的第一行文字的基线对齐。
  • stretch(默认值) : 如果弹性项未设置高度或设置为auto，将占满整个容器的高度。

如图：


9.1.6 align-content

align-content 用于修改 flex-wrap 属性的行为。类似于 align-items , 但它不是设置弹性子元素的对齐，而是设置各个行的对齐。

六个值分别表示：

  • flex-start(默认值) : 各行向容器的起点位置堆叠。
  • flex-end : 各行向容器的结束位置堆叠。
  • center : 各行向容器的中间位置堆叠。
  • space-between : 弹性项靠紧y轴的上下端，剩下的弹性项在y轴上平均分布。
  • space-around : 效果同 space-between ，但是在弹性项的纵轴与y轴的上下端之间，隔着的距离为弹性项与弹性项之间距离的一半。（实际效果是 justify-content 的垂直版）
  • stretch(默认值) : 各行会伸展占据剩余的空间。

如图：


9.2、 弹性项

容器内的弹性项可以设置以下的六个属性：

  1. order
  2. flex-grow
  3. flex-shrink
  4. flex-basis
  5. flex
  6. align-self

9.2.1 order

order 用数值来定义排列顺序，数值小的排在前面，默认值为0，可以是负值。


9.2.2 flex-grow

flex-grow 定义弹性项的放大比例，默认值是 0（当为0时，如果存在剩余空间，也不放大）。

如果所有的弹性项的 flex-grow 都为1时，它们将等分剩余空间（如果有的话）; 如果其中一个弹性项的 flex-grow 为2，而其他的弹性项的值是1时，那么它所占的空间会是值为1的两倍。


9.2.3 flex-shrink

flex-shrink 定义弹性项的缩小比例，默认值为 1，不可以是负值。

如果所有的弹性项的 flex-shrink 都为1时，当空间不足时，所有的弹性项都会缩小 ; 如果其中一个弹性项的 flex-shrink 为0，而其他的弹性项的值是1，当空间不足时，值为0的弹性项不缩小。


9.2.4 flex-basis

flex-basis 定义了在分配多余空间之前，弹性项占据x轴的空间，浏览器通过这个属性，计算x轴是否有多余的空间。默认值是auto，即弹性项本来的大小。

的值可以是一个具体的值，如：350px。


9.2.5 flex

flex 是 flex-grow 、flex-shrink 和 flex-basis 的结合。后两个属性可以不写

该属性有两个默认值

auto ==> (1,1,auto) //剩余空间时等分，空间不足时缩小

none ==> (0,0,auto) //不放大也不缩小


9.2.6 align-self

align-self 用于设置单个弹性项在y轴上的对齐方式。可以覆盖 align-items 属性，默认值是 auto ， 表示继承父元素的 align-items 。如果没有父元素，则与 stretch 相同。

以上六个值，除了 auto ，其他效果都与 align-items 相同。


使用弹性盒子实现完美的居中，只要设置 margin : auto; ，就可实现弹性项在两轴方向上完美居中。

十二、数据属性 data-*

所有以 data-* 开头的属性为 数据属性。用于存储页面或应用程序的私有自定义数据(可以定义在标签上，但是并不会在浏览器中显示出来)；

数据属性 让我们可以在所有的 HTML 元素上嵌入自定义的 data 属性的能力。存储（自定义）的数据能够被页面的 JavaScript 利用，以创建更好的用户体验（不进行 Ajax 调用或服务器端数据库查询）。

data-* 属性包括两部分：

  • 属性名 ：不包含任何大写字母，并且在前缀 data- 之后必须有至少一个字符。

  • 属性值 ：可以是任意字符串。

例（图片响应式）：

各种中断都需要正常显示图片，在PC端要用大图片，而在移动端应使用较小的图片。

1、将元素中直接设置的图片背景删除，换成两个 数据属性（如：data-image-xm=”小图路径” , data-image-lg=”大图路径”）

2、通过 JavaScript 的方式获取屏幕的宽度，判断屏幕宽度是否小于一定的值，选择使用对应的大图片或小图片。

十三、em 和 rem

13.1、 em

em 单位是相对当前容器的 font-size 来确定的，一个单位的 em = 当前容器定义的字体大小（font-size）。

13.2、rem

rem（root em） 单位是相对根容器（body）的 font-size 来确定的，如果整个网页都是通过 rem 的单位设置尺寸，只要在根容器（body）设置一个具体的 font-size 为像素单位，当这个像素值变化，整个页面所有用到 rem 的地方都会等比例变化，便于维护。
