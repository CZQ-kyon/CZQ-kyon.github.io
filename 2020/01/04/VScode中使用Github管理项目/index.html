13、环境变量

检查环境变量中是否存在 GIT_HOME ，若不存在，就新建一个。设置位置为刚刚第三步中填入的目录，这里我选择的是E盘下的GIT

二、VScode 中使用 git 管理项目

1、进入项目文件夹中

在vscode中，按

ctrl + ~

进入终端

  • 初始化 .git 仓库：

初始化完成后在文件夹中会出现一个 .git 的灰色文件夹，这是本地用来暂时存放代码的仓库


  • 查看本地仓储的状态

此时终端中会显示还未提交的文件


  • 将被跟踪的文件添加到跟踪列表

  • 添加本地GIT忽略清单文件 .gitignore（此文件用于说明忽略的文件有哪些）
    一般我们不将 node_modules文件夹 添加进去（只要package.json存在，就能重新下载node_modules）

新建 .gitignore 文件

//.gitignore
node_modules

如果不想将.js后缀的文件托管上去，在 .gitignore 文件里增加：

//.gitignore
node_modules
*.js


  • 提交被托管文件变化到本地仓储.git中

单引号里的xxx可以添加的是 你对这个提交的说明，如：’今天的第一提交’


  • 对比差异命令
    与 .git 中最后一个添加的版本进行对比

  • 查看日志

终端中会显示你历次提交的记录


  • 版本回归
    如果在编辑过程中，代码打错了，想回到上一个版本中，可以先 「查看日志」，然后通过 显示出来的 commit 前6位哈希值，进行回归
