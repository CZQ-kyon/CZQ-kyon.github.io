注意： 2.2.0+的版本里，当在组件中使用 v-for 时，key 是必须的。
当 Vue.js 用 v-for 更新已渲染过的元素列表时，它默认用“就地复用”策略。如果数据项的顺序被改变， Vue 将不是移动 DOM 元素来匹配数据项的顺序，而是简单复用此处的每个元素，也就是不改变被删节点后的节点对应的索引。

例：当把 1，2，3 中的第二个元素删除，列表变成：1，3。不会改变节点的索引。

为了给 Vue 一个提示，以便它能跟踪每个节点的身份，从而重用和重新排序现有元素，需要为每项提供一个唯一 Key 属性。

v-for 循环的时候， key 属性只能使用 number 或者 String
key 在使用的时候，必修使用 v-bind 属性绑定的形式，指定 key的值。
在组件中，使用 v-for 循环的时候，或者在一些特殊情况中，如果 v-for 有问题，必须在使用 v-for 的同时，指定唯一的 String / number 类型 ：key 值。

在表单类元素上实现数据的双向绑定。
input（radio,text,address,email…），select，checkbox，textarea

7、v-if 和 v-show指令

  • v-if 特点：每次都会重新删除或创建元素

  • v-show 特点：每次不会重新进行 DOM 的删除和创建操作，只是切换了 display：none 的样式

总结：
v-if 有较高的 切换 性能消耗
v-show 有较高的 初始渲染 消耗

如果元素涉及到频繁的切换，最好使用 v-show；
如果元素可能永远也不会被显示出来被用户看到，则推荐使用 v-if
